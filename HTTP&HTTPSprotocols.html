HTTP & HTTPS Protocols

HTTP: It stands for Hypertext Transfer Protocol. This protocol helps in transmission of data over the World Wide Web. It is located in the application layer which is the seventh layer of OSI model.

HTTPS: It stands for Hypertext Transfer Protocol Secure. This protocol helps in transmission of the data over the World Wide Web. It is located in the Transport layer which is the fourth layer of OSI model.

What is the purpose of HTTP vs HTTPS?

When transmitting data over the web “Data Security” plays a significant role. We want data to be transported securely without any breakage or being hacked. HTTP is regarded as unsecure whereas HTTPS is marked as secure. We prefer HTTPS over HTTP the reason is “Data Security” as sensitive data is sent over the web like SSN,credit card information, login credentials, license number, address etc. HTTP can be easily hacked but HTTPS is encrypted and cannot be hacked the "lock" present with the URL represents the site is secure to transmit sensitive data. Digital certification plays a key role in encryption of data being sent over the internet.
